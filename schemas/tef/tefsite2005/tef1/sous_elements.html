Qualifieurs Dublin Core utilisés comme sous-éléments XML

Dans TEF, les qualificatifs de raffinement** sont utilisés comme enfants des éléments Dublin Core qu’ils qualifient. En d’autres termes, les sous-propriétés Dublin Core (au sens sémantique) sont traitées comme sous-éléments XML (au sens syntaxique). Exemple : 

<dc.coverage>
	<dcterms.spatial/>
</dc.coverage>

Cette option permet une meilleure lisibilité des notices XML de TEF, même si cette syntaxe n’est pas recommandée par le DCMI.

Dans le cas de dc.format, cette option permet d’associer la taille d’un fichier ( dcterms.extent ) avec son format ( dcterms.medium ).
